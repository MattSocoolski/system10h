## A. DANE PRODUKTOWE — ZANIM ZACZNIESZ PROJEKTOWAĆ

### A1. Obowiązkowe dane per SKU (MUST-HAVE)

- [ ] **Nazwa produktu** — pełna, jednoznaczna (np. "Zestaw PBN z ramą 40×50 cm" nie "PBN duży")
- [ ] **Zdjęcie packshot** — białe tło, min. 2000×2000 px, PNG/TIFF
- [ ] **Zdjęcie lifestyle** — produkt w użyciu (malowanie, gotowy obraz na ścianie, event)
- [ ] **Kod EAN-13 (GTIN)** — per wariant/rozmiar. Bez EAN = buyer nawet nie otworzy katalogu
- [ ] **Wymiary produktu** — D×S×W w mm lub cm
- [ ] **Waga brutto** — per sztuka (z opakowaniem)
- [ ] **Skład zestawu** — co dokładnie jest w pudełku (dla PBN: płótno, farby, pędzle, haczyki)
- [ ] **Marka** — brand name + logo marki (ArtNapi / Adré Artó / inna)
- [ ] **Certyfikaty** — które certyfikaty dotyczą tego produktu (CE, EN 71, FSC, BSCI, REACH)

### A2. Dane logistyczne (MUST-HAVE dla retail, NICE-TO-HAVE dla dystrybucji)

- [ ] **Opakowanie zbiorcze (master carton)** — ile sztuk w kartonie
- [ ] **Wymiary kartonu** — D×S×W w cm
- [ ] **Waga kartonu** — brutto w kg
- [ ] **Kod ITF-14** — karton zbiorczy (wymagane przez sieci z EDI)
- [ ] **Paleta** — ile sztuk na palecie (warstwy × kartony)
- [ ] **Wymiary palety** — EURO 80×120 lub inna
- [ ] **Etykieta SSCC** — numer jednostki logistycznej (wymagane przez duże sieci)

### A3. Dane handlowe (OPCJONALNIE w katalogu — częściej w osobnym cenniku)

- [ ] **SRP (Suggested Retail Price)** — sugerowana cena detaliczna brutto. Buyer MUSI wiedzieć jaka marża
- [ ] **Cena B2B netto** — per prog (kartony / 1 pal / 2 pal / 3 pal). UWAGA: często OSOBNY dokument
- [ ] **MOQ** — minimum order quantity
- [ ] **Lead time** — czas realizacji (np. "24h z magazynu PL")
- [ ] **Warunki dostawy** — ALL-IN / EXW / DAP / DDP

### A4. Dane marketingowe (NICE-TO-HAVE — wzmacniają przekaz)

- [ ] **Bestsellery** — TOP 5-10 wzorów/produktów z danymi sprzedażowymi
- [ ] **Rotacja** — sell-through rate (szt/mies, trend)
- [ ] **Sezonowość** — kiedy sprzedaje się najlepiej (BTS, BN, Dzień Dziecka)
- [ ] **Recenzje / oceny** — średnia z marketplace'ów (np. ★4.7 z 2000+ opinii na Allegro)
- [ ] **Pozycja rynkowa** — "TOP 3 na Allegro w kategorii PBN"

## B. STRUKTURA KATALOGU PRODUKTOWEGO

### B1. Strony obowiązkowe (MUST-HAVE)

- [ ] **OKŁADKA** — logo, nazwa firmy, rok, tagline (1 zdanie), hero image, dane kontaktowe
- [ ] **O FIRMIE** — 1 strona max. Capability statement > historia. Kluczowe fakty: magazyn, czas dostawy, rynki, certyfikaty
- [ ] **STRONY PRODUKTOWE** — 1 kategoria = 1 strona (lub 1 spread). Packshot dominuje, dane pod spodem
- [ ] **KONTAKT + CTA** — imię KAM, email, tel, www, QR code (do kalkulatora/landing page)

### B2. Strony rekomendowane (NICE-TO-HAVE — zwiększają konwersję)

- [ ] **BESTSELLERY** — grid 8-12 produktów z miniaturkami (buyer widzi popularność)
- [ ] **PORÓWNANIE** — tabela "My vs Standardowi Dostawcy" (marża, czas dostawy, certyfikaty, stock)
- [ ] **LIFESTYLE / USE CASES** — zdjęcia produktu w akcji (event, sklep, półka, malowanie)
- [ ] **CERTYFIKATY** — dedykowana sekcja lub pasek z badgami na każdej stronie
- [ ] **MAPA DOSTAW / RYNKI** — wizualizacja zasięgu (14 EU markets)
- [ ] **PRIVATE LABEL / CUSTOM** — jeśli oferujesz (MOQ, opcje, timeline)

### B3. Czego NIE wkładać do katalogu produktowego

- [ ] ~~Długa historia firmy~~ → max 2 zdania
- [ ] ~~Pełny cennik z progami~~ → osobny dokument
- [ ] ~~Słowa "importujemy z Chin"~~ → "dostarczamy" / "magazyn w Polsce"
- [ ] ~~Zbyt wiele tekstu~~ → packshot > opis. Buyer skanuje, nie czyta
- [ ] ~~Ogólniki~~ → "wysoka jakość" zamień na "280g/m², potrójnie gruntowane, FSC certified"

## C. DESIGN I FORMAT

### C1. Format techniczny

- [ ] **Format strony:** A4 portrait (210×297 mm) — standard w EU
- [ ] **Rozdzielczość zdjęć:** min. 300 DPI (druk) / 150 DPI (email PDF)
- [ ] **Kolory:** CMYK dla druku, RGB dla ekranu. Przygotuj oba warianty
- [ ] **Bleed:** 3mm na każdą stronę (jeśli planowany druk)
- [ ] **Waga PDF:** <5 MB (wersja email), <20 MB (wersja full druk)
- [ ] **Page breaks:** @media print z page-break-after dla każdej sekcji
- [ ] **Numeracja stron:** footer z logo + nr strony

### C2. Typografia

- [ ] **Max 2 fonty:** 1 serif/display (nagłówki) + 1 sans-serif (body)
- [ ] **Hierarchia:** H1 (32-40pt) → H2 (24-28pt) → H3 (18-20pt) → Body (10-12pt)
- [ ] **Czytelność:** min. kontrast 4.5:1 (WCAG AA). Tekst ciemny na jasnym tle
- [ ] **Nie używaj:** Comic Sans, Papyrus, dekoracyjnych fontów w danych produktowych

### C3. Kolory i branding

- [ ] **Paleta barw z brand guide** — max 4-5 kolorów. Primary + Accent + Neutral + tła
- [ ] **Kolor tła per kategoria** — różne, ale spójne (jak "Dobry Pies": każda kategoria = inny pastel)
- [ ] **Logo na KAŻDEJ stronie** — mały, w narożniku (top-right lub bottom)
- [ ] **Spójność** — ten sam styl karty produktowej na wszystkich stronach

### C4. Layout stron produktowych

- [ ] **Packshot = 40-60% powierzchni** — dominujący element. Buyer patrzy na produkt, nie na tekst
- [ ] **Dane pod/obok zdjęcia** — nazwa, kluczowa spec, EAN, karton, paleta
- [ ] **Ikony > tekst** — wymiary, waga, certyfikaty jako ikony
- [ ] **White space** — nie zapychaj strony. Oddech = profesjonalizm
- [ ] **Konsekwentny grid** — te same pozycje na każdej stronie (EAN zawsze w tym samym miejscu)

## D. CONTENT I MESSAGING

### D1. Język i ton

- [ ] **B2B tone:** profesjonalny, konkretny, bez emocji. Fakty > przymiotniki
- [ ] **Buyer-centric:** nie "jesteśmy wspaniali" → "Twoja marża wyniesie 40%+"
- [ ] **Bez ujawniania łańcucha dostaw:** "Dostarczamy kompletne rozwiązania" (nie "importujemy z Chin")
- [ ] **Własna marka:** "Nasze marki: ArtNapi + Adré Artó" (nie "private label")
- [ ] **Konkret:** "280g/m², FSC, wysyłka 24h" (nie "najwyższa jakość, szybka dostawa")

### D2. Opisy produktów (max 3 zdania per produkt)

- [ ] **Zdanie 1:** Co to jest + rozmiar + wariant
- [ ] **Zdanie 2:** Kluczowy USP / wyróżnik (np. "Kompletny zestaw: płótno, farby, pędzle, haczyki")
- [ ] **Zdanie 3:** Benefit dla klienta końcowego (np. "Gotowy do malowania w 5 minut — idealny na event lub prezent")

### D3. Sekcja "O firmie" — Capability Statement (nie historia)

- [ ] **Zdanie 1:** Kim jesteśmy (1 zdanie)
- [ ] **Fakty:** Magazyn (lokalizacja), czas dostawy, liczba SKU, rynki, certyfikaty
- [ ] **Social proof:** "TOP 3 na Allegro", "X szt/mies. sprzedaży", "Y klientów B2B"
- [ ] **Certyfikaty:** badge'e FSC, BSCI, EN71, CE, REACH — wizualnie, nie tekstowo

### D4. CTA (Call to Action) — na KAŻDEJ stronie

- [ ] **Kontakt na okładce i ostatniej stronie** — email + tel + www
- [ ] **QR code** — do kalkulatora B2B / landing page / formularz zamówienia
- [ ] **"Request samples"** — próbki fizyczne to standard w branży art supplies
- [ ] **"Download full price list"** — link do cennika (oddzielny dokument)

## F. CHECKLIST PRE-FLIGHT (przed wysłaniem/drukiem)

### F1. Dane

- [ ] Wszystkie EAN-13 zweryfikowane (skan test)
- [ ] Wymiary i wagi aktualne (sprawdzone z aktualnym stanem magazynowym)
- [ ] Ceny (jeśli są) = aktualne z oferta.md (NIE z pamięci)
- [ ] SRP = aktualne z rynku (sprawdzone na Allegro/marketplace)
- [ ] Certyfikaty = te które MAMY (nie te które "planujemy uzyskać")
- [ ] Ilości w kartonie/palecie = aktualne (sprawdzone z magazynem)

### F2. Design

- [ ] Logo na każdej stronie
- [ ] Spójna typografia (max 2 fonty)
- [ ] Spójna paleta kolorów (z brand guide)
- [ ] Zdjęcia ostre, dobrze wykadrowane, białe tło czyste
- [ ] Hierarchia wizualna: packshot > nazwa > spec > EAN > logistyka
- [ ] White space — strony nie są "zapchane"
- [ ] Footer z numeracją stron + mini logo

### F3. Content

- [ ] Zero błędów ortograficznych (proofreading!)
- [ ] Zero ogólników — każde twierdzenie poparte liczbą lub faktem
- [ ] Ton spójny na wszystkich stronach
- [ ] Dane kontaktowe aktualne (email, tel, www)
- [ ] QR code działa (test skanem!)
- [ ] Brak "importujemy", "chiński", "tani"
- [ ] Brak "private label" jeśli to marka własna

### F4. Format

- [ ] PDF eksportowany z Chrome (Cmd+P → Save as PDF → A4 → Margins: None)
- [ ] Page breaks działają (każda kategoria = nowa strona)
- [ ] Zdjęcia renderują się poprawnie (nie są rozmazane/przycięte)
- [ ] Waga pliku: <5 MB (email) / <20 MB (druk)
- [ ] Test na telefonie — czy czytelny?
- [ ] Test wydruku na A4 — czy kolory/layout OK?

## G. ANTI-PATTERNS — CZEGO UNIKAĆ

### G1. "Zabójcy" katalogów B2B (z naszego doświadczenia + Deep Research)

| Anti-pattern | Dlaczego zabija | Fix |
|---|---|---|
| Za dużo tekstu, za mało zdjęć | Buyer skanuje <10 sekund. Tekst = zamknięcie | Packshot 60% powierzchni |
| Brak EAN | Buyer nie może wpisać do systemu = kosz | EAN na KAŻDYM produkcie |
| Ogólniki ("wysoka jakość") | Każdy to mówi. Zero wyróżnika | Konkret: "280g/m², FSC, 24h dispatch" |
| Jeden katalog na wszystko | Retail, eventy, WTZ mają inne potrzeby | Warianty per segment (sekcja E) |
| Ceny w katalogu | Cennik się zmienia, katalog nie | Osobny cennik. Katalog = SRP |
| "Importujemy z Chin" | Buyer widzi ryzyko, nie wartość | "Magazyn PL, 24h, ALL-IN" |
| Brak porównania z konkurencją | Buyer nie wie dlaczego MY | Tabela porównawcza |
| Logo tylko na okładce | Buyer drukuje 1 stronę — nie wie czyje | Logo na KAŻDEJ stronie |
| Brak CTA | Buyer przeczytał — i co dalej? | CTA + QR + "Request samples" |
| Zbyt wiele stron | >15 stron = nikt nie czyta | 8-12 stron (katalog), 5-7 (pitch deck) |

### G2. Feedback Nazar (marzec 2026) — konkretne błędy naszego v1

1. "Ogólny i nic nie mówi" → brak packshot'ów produktowych
2. "Tekst-heavy" → za dużo opisów, za mało zdjęć
3. "Nie widać co sprzedajesz" → brak hero image produktu per kategoria
4. **Fix w v3:** Packshoty, lifestyle, EAN, bestseller grid, porównanie, certyfikaty

## L. PSYCHOLOGIA BUYERA (z Deep Research Gemini, 20.03.2026)

### L1. Ile czasu ma buyer

- Buyer w polskiej sieci non-food dostaje **30-50 ofert/tydzień**, 100-120 emaili dziennie
- Skanuje katalog **30-60 sekund** zanim zdecyduje: spotkanie czy kosz
- Wzorzec skanowania: **Z** (druk) / **F** (digital) — dolny prawy róg pomijany przez 77% ludzi
- **Kluczowe dane (SRP, marża) muszą być w górnej 1/3 strony i lewej kolumnie**

### L2. Sekwencja skanowania (0-30 sekund)

| Czas | Co buyer sprawdza | Nasz element |
|------|-------------------|--------------|
| 0-3 sek. | Logo, jakość graficzna → "czy to serio?" | Profesjonalny packshot |
| 3-8 sek. | Hero shot + headline → "co to jest?" | Nazwa produktu + SRP |
| 8-15 sek. | **SRP + cena zakupu → kalkulacja marży** | **Margin badge (55-65%)** |
| 15-25 sek. | USP / punkt różnicujący | "Magazyn PL, 24h, FSC/BSCI" |
| 25-30 sek. | Logistyka: MOQ, lead time, certyfikaty | EAN, CE, warunki |

### L3. Instant rejection triggers

- [ ] Brak ceny/marży → buyer nie ma czasu szukać
- [ ] Słabe zdjęcia → rozmyte, niespójne, brak białego tła
- [ ] Brak EAN → nie da się wpisać do systemu
- [ ] Marża <40% → sieci non-food oczekują 40-60%
- [ ] Brak certyfikatów CE/EN71
- [ ] Duplikacja istniejącej oferty bez wyróżnika

### L4. Subject line emaila

- Personalizacja: **+46% open rate** vs bez
- Długość: **<7 słów = +65% performance**
- Formuła: `[Kategoria] + [Korzyść liczbowa] + [Nazwa Sieci]`
- Przykład: `"PBN 40×50 z ramą, marża 55% — ArtNapi dla [Empik]"`
- UNIKAĆ: "Oferta współpracy", "Nowa kolekcja", ALL CAPS

### L5. Druk vs ekran

- 55% emaili otwieranych na mobile
- Buyer drukuje 1-2 stronnicowy summary na biurowej drukarce (często B/W!)
- **Sell sheet MUSI działać w czarno-białym druku**

## M. PITCH DECK — STRUKTURA 12 SLAJDÓW

### M1. Podział czasu spotkania (15-30 min)

| Część | % czasu | Opis |
|-------|---------|------|
| Small talk | 10% | Budowanie relacji |
| Firma | 15% | 4 ikony, nie historia |
| **Produkty + marże** | **45%** | Najważniejsza część! |
| Propozycja współpracy | 15% | MOQ, warunki, next steps |
| Q&A | 15% | Pytania buyera |

### M2. 12 slajdów

1. **COVER** — logo + tagline + logo SIECI (spersonalizuj!)
2. **AGENDA** — 5-6 punktów
3. **O NAS** — 4 ikony (magazyn PL, 24h, FSC+BSCI, ALL-IN)
4. **RYNEK I TRENDY** — Google Trends PBN, dane Allegro/Amazon
5. **PROBLEM I OKAZJA** — luka w kategorii sieci + zdjęcie z ich sklepu
6. **PORTFOLIO TOP 5 SKU** — packshoty + SRP (nie cały katalog!)
7. **USP — 5 ikon** — szybkie, ikonograficzne
8. **ANALIZA MARŻY** ⭐ — SRP → Cena netto → GM% per SKU (KEY SLIDE!)
9. **PLANOGRAM / RENDERKA PÓŁKI** — moduł ArtNapi na regale 60/100/200cm
10. **WSPARCIE MARKETINGOWE** — POS, social media, eventy
11. **LOGISTYKA I WARUNKI** — MOQ, lead time, EDI
12. **PODSUMOWANIE + NEXT STEPS** — 3 argumenty + CTA

- **Appendix:** pełna lista SKU, case studies, certyfikaty
- **Cennik:** na slajdzie 8 tylko SRP + marża TOP SKU. Pełny cennik = leave-behind
